# Sector-Paper-Video Skill

## 概述

```
同花顺问财（最热板块，仅作选题锚点）
       ↓
OpenAlex（1 篇中国论文 + 摘要必读，核心要求：近一年内）
       ↓
OA 论文优先取全文（提取 Results/Methods 中的具体数据，充实文案细节）
       ↓
深度科普文案（不讲股票，只讲技术和产业）
       ↓
剪映 AI文字成片（自动填入文案，生成视频）
```

**核心原则**：板块涨跌只是选题信号，文案里不提股价、不念 IF、不数引用次数。只讲故事——这项技术是什么、为什么厉害、能改变什么产业。

**论文选择铁律**：时效第一、期刊第二。只要近一年内、中国团队主导、摘要能读懂，期刊大差不差即可。不追顶刊、不数 IF。

用户通过排名指令控制选题方向：

| 用户输入 | 行为 |
|---------|------|
| `sector-paper-video 排名第一的板块` | 取涨幅第 1 名板块 |

未指定排名时默认取第 1 名。

## 流程

### Step 1：选题——用板块定方向（~30s）

调用 hithink-sector-selector，查询同花顺问财 API：

```
query: "今日涨幅最大的行业板块"
page: 1, limit: 5
```

从返回的 `datas` 数组中按排名取对应板块。核心字段：`指数简称`（板块名）、`涨跌幅[YYYYMMDD]`。

**注意**：返回字段名每次先打印 `list(datas[0].keys())` 确认，字段名含日期后缀（如 `涨跌幅[20260618]`），不要硬编码。

如果同花顺无数据 → 换"主力资金净流入最大的板块"重试，最多 2 次。

### Step 2：关键词映射（即时）

读取 `references/sector-keyword-map.md`，将板块名映射为学术搜索关键词。优先英文关键词（OpenAlex 覆盖广），中文关键词作为知网兜底。

板块不在映射表时：英文翻译 + 常识推断，标注"（推断）"。

### Step 3：找 1 篇最佳论文（~60s）

#### 3a. OpenAlex 搜索（关键：必须用 title.search 而非 search）

❌ **错误做法**：`search=tungsten` — 返回大合作组论文（CERN、LHC 等），中国只是参与者之一

✅ **正确做法**：用 `filter` 参数组合精确筛选：

```
filter: title.search:{英文关键词1},
        authorships.institutions.country_code:CN,
        type:article,
        publication_year:{当年或上年}      ← 时效硬约束：不超过最近一年
sort: cited_by_count:desc
per_page: 10
```

**OpenAlex API 调用模板**：

```python
base = 'https://api.openalex.org/works'
params = {
    'filter': f'title.search:{keyword},authorships.institutions.country_code:CN,type:article,publication_year:{year}',
    'sort': 'cited_by_count:desc',
    'per_page': '10',
    'mailto': 'claude-code@anthropic.com'
}
```

#### 3b. 论文筛选优先级

从 Top 10 中按此顺序挑 1 篇：

1. **时效硬约束**：必须是近一年内（当年或上年），超过一年的直接跳过
2. **中国机构主导**：第一/通讯作者机构必须是中国单位（不只是参与名单里有中国机构）
3. **摘要可读性**：摘要必须能读懂核心逻辑——至少读 3 遍，读不懂就换下一篇
4. **期刊加成（锦上添花，非必须）**：Nature / Science / Nature Comms / Adv. Mater. / JACS / ACS Nano 等顶刊优先，但**不强制**。好期刊只是加分项，不是准入门槛
5. **兜底**：引用最高的那篇

如果关键词 1 无结果或结果不相关 → 换关键词 2、3 依次试。仍无结果 → 放宽到 `search=` 全文搜索兜底，取引用最高的中国论文。无中国论文 → 输出空跑记录到 content 目录。

#### 3c. 拉取摘要

用 DOI 精确查询（优先 OpenAlex，无摘要时依次尝试 Semantic Scholar、Crossref）：

```python
# 首选 OpenAlex
url = f'https://api.openalex.org/works/doi:{doi}'
# 兜底 Semantic Scholar
url = f'https://api.semanticscholar.org/graph/v1/paper/DOI:{doi}?fields=title,abstract'
# 兜底 Crossref
url = f'https://api.crossref.org/works/{doi}'
```

#### 3d. 尝试阅读全文（OA 论文优先，新增）

摘要只是骨架，**全文里的数据才是血肉**。摘要有以下不足：
- 只有结论没有推导过程，写背景科普时缺少细节
- 没有与同类技术对比的具体数字，只能泛泛说"突破了天花板"
- 没有 Methods 中的实验细节，无法解释"怎么做到的"

**操作流程**：

1. 检查 `open_access.is_oa` — OA 论文直接尝试取全文
2. 用 WebFetch 抓取论文 HTML 页面（优先出版社官网，其次 PubMed Central、arXiv、ResearchGate）
3. 重点提取：
   - **Introduction**：领域背景、现有技术瓶颈的具体数据
   - **Results**：关键性能数字、对比实验数据、图表标题
   - **Methods**：工艺细节（如"低温共烧""高熵设计"怎么做的）
   - **Discussion**：作者自己说的产业前景
4. 将这些细节填入文案的「背景科普」和「这次突破」段落，用具体数字替代泛泛描述

**注意**：
- 外网（nature.com、ncbi.nlm.nih.gov 等）可能被企业网络限制，此时退回摘要写文案，标注"仅摘要"
- PDF 全文无法直接抓取时，尝试论文的 HTML 版本
- 千万不要因为读全文而卡住流程——**摘要足够就继续往下走**，全文是锦上添花

**摘要必须读懂再进入 Step 4**。读三遍还不懂 → 换下一篇，不硬写。

## 文案结构（严格遵循）

```
[开场钩子] XX领域传来一条重磅消息，我国科研团队在{什么平台上/用什么方法}
实现了{什么}，{一个震撼数字或类比}。

[认知门槛共情] 很多人一看这个消息有点懵，{名词}又长又难懂，然后就顺手划走了。
注意里面两个关键词，一个是{关键词1}，一个是{数量级/关键词2}，对你没听错，
不是{小数字}，不是{中数字}，是{大数字}，这个数量级就足以说明这个突破的厉害程度。

[破除质疑] 可能有人觉得这就是个实验室玩意，离产业化远得很。但你仔细了解后
就会发现，这不是凑论文的花架子，它精准戳破了{领域}几十年的技术天花板，
接下来会直接改写{行业A}、{行业B}两大千亿赛道的游戏规则。

[转入正题] 别着急，接下来我花 3 分钟时间把其中核心逻辑给大家一次性讲透。

[背景科普] 先给大家补个背景。{技术名}不是新鲜东西，{起源/原理}拿过诺贝尔奖/
是经典理论，但问世几十年始终没走出实验室。核心就是三个天生短板解不开：
第一，{短板1}。第二，{短板2}。第三，{短板3}。
这三个问题卡了几十年，导致{技术}始终是{小众工具/实验室概念}，市场规模常年只有{小数字}，
撑不起产业行情。

[这次突破] 但这次的{新技术名}相当于换了整条技术路线，把{旧思路}改成了{新思路}，
三个突破正好精准解决上面三个痛点：
第一，{突破1 + 震撼数字}。{1-2句展开}。
第二，{突破2}。{1-2句展开}。
第三，{突破3}。{1-2句展开}。

[产业落地] 技术突破只是 0 到 1，真正的价值要看 1 到 N 的落地。
这个技术最狠的地方是每一个优势都精准对应了产业的真痛点，不是为了炫技而炫技。

[赛道一] 第一个赛道是{行业名}。{市场规模}。{具体痛点}。
这个{技术}能{解决方案}，相当于{类比}。这个场景一旦落地，就是{数字}的新增市场。

[赛道二] 第二个赛道是{行业名}。{具体痛点}。{技术如何解决}。
对{相关产业链}是底层工具级的利好。

[赛道三（可选）] 第三个赛道是{行业名}。{简述}。

[商业化时间] 很多人会问，离商业化还有多远？客观说大规模{应用场景}还要{时间}。
但{近期可落地的}会非常快。机会会沿着产业链从上到下逐步兑现：
最上游的{上游环节}技术成熟度最高，会最先受益；
中游的{中游环节}有现成客户渠道的，能最快把技术转化成商用产品；
下游的{下游环节}则是长期享受工具升级带来的红利。

[国家站位] 而且最关键的是，这是我们全球首创的技术，底层专利握在自己手里，
不存在卡脖子的问题。过去我们是高端{领域}的进口国，这次我们有机会成为全球{领域}的输出方。
以前我们说硬科技总讲国产替代，别人有什么我们追什么；
现在我们开始做出别人没有的技术，开始定义新的技术路线，这才是真正的硬科技含金量。

[风控收尾] 当然任何技术从实验室到产业化都有不确定性，落地进度、商业化成本都可能有变数，
但大方向是确定的。{一句话总结行业趋势}。底层工具的突破从来都是产业升级的先行信号。

好了，本文仅为产业逻辑科普，不构成任何投资建议。股市有风险，入市需谨慎。
关注我带你看懂财经热点背后的底层逻辑。
```

## 写作铁律

1. **不讲股票**：文案里不出现"涨了 X%""板块""主力资金"等任何股票术语。板块只是选题工具。
2. **不讲 IF**：不出现"影响因子""IF 40.2""JCR 一区"。期刊名自然带过即可（如"国际能源顶刊《Energy & Environmental Science》"）。
3. **不数引用**：不出现"被引 56 次"。
4. **一定有震撼数字**：找到论文里最震撼的一个数量级——10 万倍、100 倍、首次、全球首创。用"对你没听错，不是 X 倍，不是 Y 倍，是 Z 倍"的句式强化。
5. **一定有背景科普**：假设观众完全不懂这个领域。从最基础的概念讲起，用"先给大家补个背景"引入。
6. **一定有产业落地**：至少讲 2 个具体的产业应用场景，每个都点出市场规模和痛点。
7. **一定有国家站位**：强调"全球首创""底层专利在自己手里""从进口到输出"。
8. **一定有风控**：结尾必须带"不构成任何投资建议"。
9. **优先用全文数据**：OA 论文优先读全文取具体数字（Results 里的对比数据、Methods 里的工艺细节），摘要只是底线。全文读不到则退回用摘要，不卡流程。
10. **长度**：1800-2500 字，适合 3 分钟口播。

### Step 5：剪映 AI文字成片（~30s）

文案写完后，用 Python 创建输出目录并调用 `jianying_ai_video.py`：

**⚠️ 重要**：输出路径含中文，Windows bash 下 mkdir 会乱码。统一用 Python 处理路径创建和脚本调用：

```python
import os, datetime, subprocess

now = datetime.datetime.now()
filename = f"{now.strftime('%Y%m%d_%H%M')}_{sector_name}.md"
out_dir = r"C:\E\coding\influencer\content"
os.makedirs(out_dir, exist_ok=True)
filepath = os.path.join(out_dir, filename)

# 写入文案后调用剪映脚本
# 脚本路径：{SKILL_BASE}/scripts/jianying_ai_video.py
# 将 {SKILL_BASE} 替换为当前 skill 的安装根目录
skill_base = r"{SKILL_BASE}"
script = os.path.join(skill_base, "scripts", "jianying_ai_video.py")
subprocess.run(["python", script, "--file", filepath])
```

脚本工作流程（已内置自动启动）：
1. 探测剪映窗口 → 未找到则自动探测安装路径并启动
2. 等待窗口出现并完成初始化（最长 30s）
3. 若在首页 → 关闭启动弹窗（先试旧版坐标 `660,550`，弹窗未消则自动换新版坐标 `870,163`）→ 点击"AI文字成片"入口
4. 在输入框中填入文案正文（自动从 MD 文件中提取 `## 文案` 后的内容）
5. 点击生成按钮 → 点击导出

**剪映窗口要求**：不能最小化。坐标基于标准窗口尺寸（首页 1168×780，AI页 1440×812），最大化时需先还原。弹窗有关闭按钮新旧两种样式，脚本自动检测并切换。

## 搜索踩坑记录

### 2026-06-21 实测 #1（钨 / 排名第1板块）

1. **`search=tungsten` 宽泛搜索不可用**：OpenAlex 的 `search` 参数做全文/全字段匹配，高引排序下返回的全是 CERN、LHC 等大型国际合作组的物理学论文，中国机构仅作为数百个参与单位之一出现。**必须用 `title.search` + `authorships.institutions.country_code:CN` 组合**。

2. **年份过滤必要**：不加 `publication_year` 过滤会返回 2008 年的 CMS 论文（5453 引用），淹没了 2024 年的真正钨材料研究。

3. **关键词顺序重要**：优先用 mapping 表中的第一个英文关键词（最核心的）。如 `tungsten` 而非 `WS2` — 核心词覆盖最广，太具体的词可能零结果。

4. **Windows bash + 中文路径 = 乱码**：创建输出目录和文件时，始终用 Python 的 `os.makedirs()` + `open(encoding='utf-8')`，不要用 bash 的 mkdir 或重定向。

5. **同花顺字段名含日期**：每次请求先 print keys，确认 `涨跌幅[YYYYMMDD]` 的实际键名再取数。

### 2026-06-21 实测 #2（磨具磨料 / 排名第3板块）

1. **板块不在映射表不可怕**：磨具磨料 → diamond → 找到了华科的金刚石荧光膜 Advanced Materials 论文。常识推断足够。

2. **Nature 论文摘要缺失普遍**：Nature 系列在 OpenAlex / Semantic Scholar / Crossref 三个 API 都可能没有摘要。三处都无摘要 → 直接跳过换下一篇。

3. **顶刊不是必须**：华科 Advanced Materials 论文虽然不是 Nature/Science，但故事完整、数字震撼、产业逻辑清晰。**时效 + 可读性 > 期刊档次**。

### 2026-06-21 实测 #3（非金属材料 / 排名第4板块）

1. **关键词多轮尝试正常**：ceramic → boron nitride → carbon fiber → 最终在 `ceramic` 搜索结果中找到清华 Nature Comms BaTiO₃ 论文。换了 3 个关键词才找到合适的，这是正常流程，不是失败。

2. **综述论文要跳过**：Materials Science and Engineering R: Reports 的 Si 陶瓷涂层论文（148 引用）是综述，没有单一突破数据点，不适合"这次突破"叙事结构。Carbon Energy 的木质素碳纤维论文也是综述。**筛选时检查 `type:article` 还不够，标题含 "review""progress""advances" 的也要跳过**。

3. **同类题材要错开**：清华 BaTiO₃ 高熵陶瓷电容和钨篇的 TTB 钨青铜陶瓷电容都是"高熵+陶瓷储能"，但体系不同（BaTiO₃ vs TTB）、数字不同（10.9 vs 8.9 J/cm³、260°C vs 180°C、10⁶ vs 10⁵ 循环）、故事角度不同（经典材料改造 vs 新型结构设计），观众不会觉得重复。**连续多篇同类题材时，确保核心数字和故事角度有明显差异**。

4. **摘要数据即够用**：本次仅用摘要就写出了包含具体数字（10.9 J/cm³、-50~260°C、10⁶ 循环、720 kV/cm）的完整文案。如果能拿到全文，背景科普中的"三个短板"可以写得更精确（比如引用 Introduction 中的传统 BaTiO₃ 具体性能数字做对比），但摘要已经提供了足够的故事主线。
